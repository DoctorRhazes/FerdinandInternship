                        I want to note that tips 2 to 16, 18 and 20 are _IPython_ functionalities. IPython is not Jupyter, and IPython is still living and going well. Saying that these are Jupyter tips (in particular the magics commands) is adding to user confusion about what is Jupyter vs what is IPython and indirectly put more pressure on the devs who will receive more miss-guided questions [Speaking from experience].

                        In particular all of the magics are available in IPython even when not using jupyter notebook and are not available in Jupyter notebook when using the non-python kernels.

                        Thanks for gathering these tips, looking forward to the next batch, and hoping to get the Jupyter vs IPython mentioned a bit more.

                                Matthias Bussonnier Carlos Gomez • 7 months ago

                                No, there is no definitive guide. But the distinction should be pretty easy. IPython feature are all the feature that are kernel-side and available only using the Python kernel. Usually just as yourself "Would it work if I were to use haskell". For example 22 (multi cursor) is a definitive "Yes". Regardless of the language you write in you can get multiple cursors, so it's a Jupyter Notebook feature. 14 (Pdb aka Python DeBugger) the answer is most likely "No, Pdb make no sens with haskell" so it's an IPython feature. After there are some feature that are "in between" and are neither obviously IPython nor obviously Jupyter, then it's a question of historical reason and better understanding what's involved under the hood. Conveying that is still hard as Jupyter is (still) often deeply associated with IPython in many people mind.

                                      Regarding tip 3, you can also press Shif+Tab as you're writing a function to have a little window pop up with a small bit of the documentation for whatever you're accessing (be it a module, class, function...). If you insist pressing Tab a few more times (usually it's 3 or 4 for me, it loads the whole documentation in a separate window at the bottom of the Notebook).
